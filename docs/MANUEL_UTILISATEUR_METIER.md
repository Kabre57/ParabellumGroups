# Manuel utilisateur métier

## 1. À quoi sert l'application

Parabellum Groups est un ERP qui centralise :

- la prospection commerciale ;
- la gestion client ;
- les devis et la facturation ;
- les opérations techniques ;
- les projets ;
- les achats et le stock ;
- les ressources humaines ;
- la communication interne.

Chaque utilisateur ne voit que les modules autorisés par son rôle.

## 2. Démarrage rapide

### Connexion

1. Ouvrir l'écran de connexion.
2. Saisir l'adresse email et le mot de passe.
3. Valider.
4. Une fois connecté, utiliser la barre latérale pour accéder aux modules disponibles.

### Navigation

Les modules sont regroupés par domaine :

- Tableau de bord
- Commercial
- CRM
- Facturation
- Services techniques
- Projets
- Achats & logistique
- Ressources humaines
- Communication

## 3. Les modules métier

## 3.1 Tableau de bord

### Objectif

Suivre rapidement l'activité globale.

### Ce que l'on trouve

- chiffre d'affaires ;
- clients actifs ;
- projets en cours ;
- interventions actives ;
- alertes stock ;
- activité récente.

### Utilisation recommandée

- ouvrir ce module en début de journée ;
- vérifier les alertes ;
- utiliser les raccourcis pour accéder au bon service.

## 3.2 Commercial

### Objectif

Transformer les prospects en opportunités, puis en clients.

### Écrans utiles

- `Prospection`
- `Pipeline commercial`
- `Devis et propositions`

### Actions principales

- créer un prospect ;
- suivre son étape d'avancement ;
- prioriser les dossiers ;
- transformer un besoin en opportunité ;
- consulter les devis envoyés.

### Bon usage

- renseigner systématiquement le contact principal ;
- mettre à jour l'étape après chaque échange ;
- utiliser les statuts de devis pour garder une vision fiable du pipe commercial.

## 3.3 CRM

### Objectif

Centraliser la connaissance client.

### Sous-modules

- Clients
- Types de clients
- Contacts
- Adresses
- Contrats
- Documents
- Interactions
- Opportunités
- Rapports
- Campagnes email

### Actions principales

- créer une fiche client ;
- enregistrer les contacts et adresses ;
- suivre les contrats ;
- stocker les documents importants ;
- enregistrer appels, emails et réunions ;
- gérer les opportunités commerciales.

### Bon usage

- créer une fiche client unique par société ;
- conserver les documents dans le CRM pour éviter les doublons ;
- journaliser chaque interaction importante.

## 3.4 Facturation

### Objectif

Produire les devis, les factures et suivre les paiements.

### Sous-modules

- Tableau de bord facturation
- Factures
- Paiements
- Devis

### Actions principales

- créer une facture ;
- vérifier la date d'échéance ;
- suivre les factures en retard ;
- enregistrer un paiement ;
- consulter les devis liés à l'activité commerciale.

### Bon usage

- vérifier le client avant émission ;
- garder les statuts à jour ;
- enregistrer les paiements dès réception.

## 3.5 Services techniques

### Objectif

Planifier et exécuter les opérations terrain.

### Sous-modules

- Spécialités
- Techniciens
- Missions
- Interventions
- Ordres de mission
- Rapports d'intervention
- Matériel

### Actions principales

- gérer les spécialités et les techniciens ;
- créer une mission ;
- planifier les interventions ;
- affecter les techniciens ;
- préparer les ordres de mission ;
- saisir le rapport d'intervention ;
- suivre le matériel utilisé.

### Bon usage

- créer la mission avant les interventions ;
- affecter les techniciens avant d'émettre l'ordre de mission ;
- produire le rapport dès la fin d'intervention ;
- vérifier le matériel disponible avant départ.

## 3.6 Projets

### Objectif

Piloter les projets, tâches et jalons.

### Sous-modules

- Projets
- Tâches
- Jalons
- Planning Gantt

### Actions principales

- créer un projet ;
- affecter un manager ;
- créer des tâches ;
- assigner les contributeurs ;
- définir des jalons ;
- suivre les échéances.

### Bon usage

- rattacher chaque tâche à un projet ;
- utiliser les jalons pour les livrables majeurs ;
- clôturer les tâches terminées pour garder un planning lisible.

## 3.7 Achats et logistique

### Objectif

Gérer les fournisseurs, commandes, réceptions et stock.

### Sous-modules

- Fournisseurs
- Commandes d'achat
- Réceptions
- Catalogue produits
- Stock

### Actions principales

- créer les fournisseurs ;
- saisir les commandes ;
- enregistrer les réceptions ;
- suivre le stock et les alertes ;
- corriger les écarts d'inventaire.

### Bon usage

- utiliser l'audit en cas d'écart physique.

## 3.8 Ressources humaines

### Objectif

Suivre les effectifs et les opérations RH courantes.

### Sous-modules

- Contrats
- Congés
- Paie
- Prêts et avances

- suivre les prêts et avances ;

## 3.9 Communication

### Objectif

### Sous-modules

- envoyer un message interne ;
- répondre à une conversation ;

- créer une campagne email ;

### Bon usage

- privilégier les notifications pour les alertes rapides ;
- utiliser les campagnes pour les communications groupées ;
- marquer comme lu pour garder une boîte propre.

## 4. Parcours métier recommandés

## 4.1 Du prospect à la facture

1. Créer le prospect dans `Commercial`.
2. Faire avancer l'étape de prospection.
3. Créer ou mettre à jour l'opportunité dans `CRM`.
4. Créer le client si la relation est validée.
5. Émettre le devis.
6. Transformer la prestation en facture.
7. Enregistrer le paiement.

## 4.2 D'une demande client à l'intervention terrain

1. Créer ou retrouver le client dans `CRM`.
2. Ouvrir la mission dans `Technique`.
3. Ajouter les interventions prévues.
4. Affecter les techniciens.
5. Générer l'ordre de mission.
6. Réaliser l'intervention.
7. Rédiger le rapport d'intervention.

## 4.3 D'un besoin interne à la réception stock

1. Vérifier le fournisseur.
2. Créer la commande d'achat.
3. Suivre le statut de la commande.
4. Enregistrer la réception.
5. Vérifier la mise à jour du stock.
6. Lancer un audit si nécessaire.

## 5. Bonnes pratiques

- Toujours vérifier le client, l'employé ou le fournisseur avant de créer une nouvelle fiche.
- Mettre à jour les statuts dès qu'une étape métier change.
- Utiliser les recherches et filtres avant de dupliquer une information.
- Produire les rapports d'intervention et validations au plus près du terrain.
- Contrôler régulièrement les alertes stock et les échéances de contrat.

## 6. Limites à connaître

Certaines fonctions sont encore en cours de finition :

- quelques actions PDF ou email en facturation ;
- certaines vues RH avancées ;
- certains écrans secondaires non visibles dans la navigation principale.
