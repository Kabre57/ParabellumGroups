# Support de formation par profil utilisateur

## 1. Objectif

Ce support transforme le manuel métier en plan de formation opérationnel par profil.

Chaque fiche profil contient :

- les objectifs pédagogiques ;
- les modules à maîtriser ;
- le parcours de formation ;
- les exercices pratiques ;
- les critères de validation.

## 2. Tronc commun

### Public

Tous les utilisateurs.

### Durée recommandée

- 45 à 60 minutes.

### Objectifs

- se connecter ;
- comprendre la navigation ;
- retrouver les modules autorisés ;
- utiliser les filtres, recherches et statuts ;
- consulter les notifications.

### Exercices

1. Se connecter.
2. Ouvrir le tableau de bord.
3. Identifier trois indicateurs utiles.
4. Ouvrir un module métier autorisé.
5. Revenir au dashboard puis consulter les notifications.

### Validation

- l'utilisateur navigue sans assistance ;
- il sait retrouver un dossier par recherche ;
- il comprend le rôle des statuts.

## 3. Profil direction

### Modules cibles

- Tableau de bord
- Analytics
- CRM
- Facturation
- Journal d'audit

### Objectifs

- suivre la performance globale ;
- repérer les retards et alertes ;
- lire les indicateurs client, financier et technique ;
- contrôler les événements d'administration sensibles.

### Séquence recommandée

1. Lecture du dashboard global.
2. Lecture des analytics par domaine.
3. Contrôle des factures en retard.
4. Contrôle des top clients.
5. Lecture du journal d'audit.

### Cas pratique

- identifier les trois priorités opérationnelles du jour à partir du dashboard ;
- détecter un risque financier et un risque stock ;
- retrouver l'événement d'audit lié à une modification de permissions.

### Validation

- le participant sait interpréter les vues de synthèse ;
- il sait relier une alerte à un module d'action.

## 4. Profil commercial

### Modules cibles

- Commercial
- CRM
- Devis
- Campagnes email

### Objectifs

- créer et qualifier un prospect ;
- faire vivre le pipeline ;
- convertir en opportunité ou client ;
- suivre les devis ;
- préparer une campagne commerciale.

### Séquence recommandée

1. Créer un prospect.
2. Mettre à jour son étape et sa priorité.
3. Ouvrir le pipeline et lire la valeur commerciale.
4. Créer ou mettre à jour l'opportunité dans le CRM.
5. Consulter les devis.
6. Préparer une campagne email simple.

### Cas pratique

- créer un prospect ;
- le faire passer à l'étape `proposal` ;
- retrouver ou créer le client ;
- vérifier l'existence d'un devis associé.

### Validation

- le participant sait maintenir le pipeline à jour ;
- il ne duplique pas les fiches client ;
- il sait relancer via les bons écrans.

## 5. Profil CRM / service client

### Modules cibles

- Clients
- Contacts
- Adresses
- Contrats
- Documents
- Interactions
- Opportunités

### Objectifs

- maintenir une base client propre ;
- suivre les documents et contrats ;
- historiser les échanges ;
- rendre la donnée exploitable par les autres services.

### Séquence recommandée

1. Créer un client.
2. Ajouter un contact.
3. Ajouter une adresse principale.
4. Créer un contrat.
5. Déposer un document.
6. Saisir une interaction.

### Cas pratique

- créer une nouvelle fiche client complète ;
- ajouter un contact et un document ;
- saisir une interaction téléphonique ;
- changer le statut d'un contrat.

### Validation

- le dossier client est complet ;
- les informations sont retrouvables ;
- les autres services peuvent exploiter la donnée.

## 6. Profil technique

### Modules cibles

- Spécialités
- Techniciens
- Missions
- Interventions
- Ordres de mission
- Rapports
- Matériel

### Objectifs

- préparer une intervention terrain de bout en bout ;
- affecter les bonnes ressources ;
- tracer l'exécution ;
- produire le rapport final.

### Séquence recommandée

1. Vérifier spécialités et techniciens.
2. Créer une mission.
3. Ajouter une intervention.
4. Affecter les techniciens.
5. Générer l'ordre de mission.
6. Produire le rapport.
7. Vérifier l'impact matériel.

### Cas pratique

- créer une mission client ;
- affecter un technicien ;
- générer un ordre de mission ;
- clôturer avec un rapport d'intervention.

### Validation

- la mission est complète ;
- l'ordre de mission est généré ;
- le rapport final est consultable.

## 7. Profil chef de projet

### Modules cibles

- Projets
- Tâches
- Jalons
- Planning Gantt

### Objectifs

- créer un projet ;
- structurer le travail ;
- suivre les jalons et la charge ;
- lire le planning global.

### Séquence recommandée

1. Créer un projet.
2. Définir le budget et le manager.
3. Créer plusieurs tâches.
4. Assigner les contributeurs.
5. Ajouter un jalon.
6. Lire le planning Gantt.

### Cas pratique

- créer un projet avec deux tâches et un jalon ;
- marquer une tâche comme terminée ;
- marquer un jalon comme atteint.

### Validation

- le participant sait structurer et suivre son projet ;
- il utilise correctement les statuts.

## 8. Profil achats / logistique

### Modules cibles

- Fournisseurs
- Commandes d'achat
- Réceptions
- Produits
- Stock
- Audit stock

### Objectifs

- gérer un achat du besoin jusqu'à l'entrée en stock ;
- suivre les fournisseurs ;
- éviter les écarts de stock.

### Séquence recommandée

1. Créer ou vérifier le fournisseur.
2. Créer la commande.
3. Consulter son statut.
4. Enregistrer la réception.
5. Vérifier le stock.
6. Corriger un écart via l'audit si besoin.

### Cas pratique

- créer une commande fournisseur ;
- réceptionner une ligne ;
- vérifier la mise à jour du stock ;
- lancer un ajustement d'audit.

### Validation

- la commande et la réception sont cohérentes ;
- le stock final est compréhensible.

## 9. Profil RH

### Modules cibles

- Employés
- Contrats RH
- Congés
- Paie
- Prêts et avances

### Objectifs

- maintenir les fiches RH ;
- gérer les demandes de congés ;
- générer la paie du mois ;
- suivre les prêts.

### Séquence recommandée

1. Créer ou mettre à jour une fiche employé.
2. Vérifier le contrat.
3. Créer une demande de congé.
4. Approuver ou rejeter.
5. Générer la paie.
6. Enregistrer un prêt ou une avance.

### Cas pratique

- créer un employé ;
- enregistrer un congé ;
- générer la paie du mois ;
- créer un prêt avec mensualité.

### Validation

- le participant maîtrise les écrans RH essentiels ;
- il comprend les dépendances entre contrat, congé et paie.

## 10. Profil administration

### Modules cibles

- Utilisateurs
- Rôles
- Permissions
- Services
- Workflow d'approbation
- Audit logs

### Objectifs

- créer un compte et l'affecter ;
- structurer les rôles ;
- gérer les exceptions de permissions ;
- tracer les changements sensibles.

### Séquence recommandée

1. Créer un service.
2. Créer un rôle personnalisé.
3. Créer un utilisateur.
4. Affecter rôle et service.
5. Gérer une surcharge de permissions.
6. Traiter une demande dans le workflow.
7. Vérifier le journal d'audit.

### Cas pratique

- créer un compte pour un nouveau manager ;
- lui affecter un rôle ;
- ajouter une permission spécifique ;
- retrouver la trace dans l'audit.

### Validation

- l'administrateur sait expliquer la différence entre rôle et override ;
- il sait diagnostiquer un problème d'accès.

## 11. Format conseillé pour les sessions

### Session initiale

- tronc commun ;
- atelier par profil ;
- cas pratique métier ;
- validation finale.

### Session de recyclage

- 30 minutes de rappel ;
- nouveautés ;
- erreurs fréquentes ;
- revue de cas réels.

## 12. Livrables à remettre après formation

- le manuel métier ;
- la version PDF-ready ;
- une fiche mémo par profil ;
- la liste des bonnes pratiques ;
- un contact de support interne.
